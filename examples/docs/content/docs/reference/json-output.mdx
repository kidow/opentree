# JSON output

Most operational commands support `--json`, including `init`, `validate`, `build`, `dev`, `deploy`, `doctor`, `import links`, `schema`, `prompt`, `config show`, edit commands, Vercel commands, and `completion`.

## Common fields

- `command`
- `stage`
- `message`
- `ok`
- `issues`
- `result`

Mutating commands that support `--dry-run` also include a top-level `dryRun` boolean and use the `dry-run` stage when the command succeeds without side effects.

## Stage meanings

- `args`: argument parsing or usage failure
- `load`: config loading or startup failure
- `validate`: schema validation failure
- `preflight`: readiness failure before mutation
- `auth`: external authentication check
- `link`: reusable link inspection or linking failure
- `build`: static output generation
- `deploy`: remote deployment handoff
- `dry-run`: validated preview with no side effects

## Compatibility guideline

Treat the common top-level keys as stable, and treat command-specific `result` payloads as additive over time.
