Your entire site is one file: `opentree.config.json`. The app reads and writes it for you, but you can also edit it directly with any text editor — useful for bulk changes, version control, or scripting.

## File location

The file lives wherever you opened or created your project. Open the file's folder from **Settings → Project → Show in Finder**.

## Top-level shape

```json
{
  "schemaVersion": 14,
  "profile": { "name": "Your Name", "bio": "Your bio" },
  "blocks": [],
  "theme": {
    "accentColor": "#16a34a",
    "backgroundColor": "#ffffff",
    "textColor": "#0a0a0a"
  },
  "siteUrl": "https://example.com",
  "domain": "links.example.com",
  "connections": ["vercel"],
  "analytics": null,
  "schedules": {},
  "seo": null,
  "locale": "en",
  "localeVariants": []
}
```

| Field | Type | Required | See |
|-------|------|----------|-----|
| `schemaVersion`   | `number` | yes | Auto-managed. Don't edit. Current: `14`. |
| `profile`         | `object` | yes | [Profile](#profile) |
| `blocks`          | `array`  | yes | [Block types](/docs/blocks) — 20 block types |
| `theme`           | `object` | yes | [Design](/docs/design) — colors, button styles, layout, backgrounds, fonts, custom CSS |
| `siteUrl`         | `string` | no  | Canonical URL set on the rendered page. |
| `domain`          | `string` | no  | Custom domain attached via Domain settings. See [Custom domains](/docs/domains). |
| `connections`     | `string[]` | no | Names of connected providers. Token values are not stored here — they live in the OS Keychain. |
| `analytics`       | `object \| null` | no | [Analytics](/docs/analytics) — Plausible config. |
| `schedules`       | `object` | no | Map of `blockId → { publishAt, unpublishAt }`. See [Scheduling](/docs/scheduling). |
| `seo`             | `object \| null` | no | [SEO](/docs/seo) — title, description, OG image. |
| `locale`          | `string` | no  | Page language code. See [i18n](/docs/i18n). |
| `localeVariants`  | `array`  | no  | Multi-locale builds. See [i18n](/docs/i18n). |

## `profile`

Header information shown above your blocks.

```json
{
  "name": "Your Name",
  "bio": "Short description shown under your name",
  "avatarUrl": "assets/avatar.png"
}
```

| Field | Type | Required |
|-------|------|----------|
| `name`      | `string` | yes |
| `bio`       | `string` | no  |
| `avatarUrl` | `string` | no — relative path inside the project, usually `assets/...` |

## `blocks`

Ordered array of content blocks. The renderer outputs blocks top-to-bottom.

Every block has at minimum:

```json
{ "id": "abc123", "type": "link", "enabled": true }
```

| Field | Type | Required |
|-------|------|----------|
| `id`      | `string`  | yes — stable unique id |
| `type`    | `string`  | yes — see [Block types](/docs/blocks) |
| `enabled` | `boolean` | yes — disabled blocks are kept but not rendered |

Per-type fields are documented at [Block types](/docs/blocks).

## `theme`

Visual customization for the rendered site. Only `accentColor`, `backgroundColor`, and `textColor` are required — all other fields are optional with sensible defaults.

```json
{
  "accentColor": "#16a34a",
  "backgroundColor": "#ffffff",
  "textColor": "#0a0a0a",
  "buttonStyle": "outline",
  "layout": "classic"
}
```

Full reference (button styles, layout variants, backgrounds, fonts, custom CSS) at [Design](/docs/design).

## Editing by hand

Two rules:

1. **Don't change `schemaVersion`.** The app upgrades it automatically when its format changes; bumping it manually can corrupt the file.
2. **Keep `id` values unique.** Block reorders and schedules depend on `id`; collisions break drag-and-drop and break schedule mapping.

Otherwise the file is plain JSON — edit it in any text editor, save, and the app will pick up the changes when you switch back.
