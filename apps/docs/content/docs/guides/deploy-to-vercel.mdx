# Deploy to Vercel

## What this page solves

You want the generated `dist/` bundle deployed to Vercel with the CLI preflight checks turned on.

## Prerequisites

- `vercel` CLI installed globally
- `vercel login` completed
- `siteUrl` configured in `opentree.config.json`
- reusable Vercel project link created at the repository root

## Recommended flow

```bash title="Terminal"
npm install -g vercel
vercel login
opentree site set --url "https://links.example.com"
opentree vercel link
opentree deploy --prod
```

## What the CLI does

1. validates the config
2. confirms `siteUrl` exists
3. checks Vercel authentication and local link status
4. runs `build`
5. syncs the project link into `dist/.vercel/project.json`
6. runs the Vercel deploy command against `dist/`

## Common failure

If `siteUrl` is empty, deploy stops at preflight and tells you to run `opentree site set --url <your-production-url>`.
