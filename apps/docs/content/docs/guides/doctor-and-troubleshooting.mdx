# Doctor and troubleshooting

## What this page solves

You want a single command that explains what is broken before you try another deploy.

## Run the checks

```bash title="Terminal"
opentree doctor
opentree doctor --json
```

Doctor focuses on:

- config file presence
- valid JSON parsing
- schema validation
- `siteUrl` readiness
- Vercel CLI installation
- Vercel authentication state
- local Vercel project link

## When to use it

- after cloning an existing project
- before the first deploy
- when `deploy` fails during preflight

## First things to verify manually

1. `opentree.config.json` exists at the project root
2. `siteUrl` is set to the production domain
3. `vercel login` has been completed
4. `.vercel/project.json` exists after `opentree vercel link`
